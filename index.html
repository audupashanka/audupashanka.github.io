gdauth.api module

gdauth.api.create_permission(fid, role, user)

This function will grant permission to a file/folder based on the type of role and the type of users provided.

Parameters

  • fid (str) – file/folder id.

  • role (str) – type of permissions (reader/writer/commenter).

  • user (list) – can be instance of user(individual user email), group(group of email ids).

Returns

None

gdauth.api.list_files(fid=None)

This function will list files/folder based on the user signed in.

Parameters

fid (str) – file/folder id when given will list the sub files and folders, if None then list the files/folders under the root directory

Returns

list

gdauth.api.list_permission(fid)

This function will return list of permissions to a file/folder based on the id.

Parameters

fid (str) – file/folder id

Returns

list

gdauth.api.update_permission(fid, permission, chg)

This function will update existing permissions to a file/folder.

Parameters

  • fid (str) – file/folder id

  • permission (dict) –

    The type of permission already set. Ex.

    {
'type': 'user',
'role': 'writer',
'emailAddress': 'user@example.com'
}
...

  • chg (dict) – type of change required(add user, change role, change type)

Returns

None

CLI Help

CLI Tool for gdauth

usage: gdauth [-h] [--version] {deploy,fetch} ...

Named Arguments

--version

Installed version of gdauth

Sub-commands:

deploy

Uploads the files and deploy the permission

gdauth deploy [-h] [--folder FOLDER] [--recursive] [--file FILE_LOC FILE_DRIVE]
Named Arguments
--folder

Directory with similar file structure as GDrive with permission files, assumes filename as .gdauth

--recursive

Recursively set permissions for the folder and its sub folders, to be used with –folder

Default: False

--file

Local file path followed by path in GDrive

fetch

Get the permissions files

gdauth fetch [-h] [--folder FOLDER] [--recursive] [--file FILE_DRIVE FILE_LOC]
Named Arguments
--folder

Folder with similar directory structure as GDrive to get all permission files, assumes filename as .gdauth

--recursive

Recursively get permissions file for the folder and its sub folders

Default: False

--file

Path of file in drive followed by the local path to fetch to
